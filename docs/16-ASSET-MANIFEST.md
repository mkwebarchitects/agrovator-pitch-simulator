## Licence and provenance gates

- No tracked `LICENSE*`, `COPYING*`, or `NOTICE*` file exists. A release authority must supply or approve the repository/code/content licence before distribution.
- The art records describe the OpenAI output terms applicable to the generating account, but no terms-version snapshot or generating-account authority evidence is archived. Human creative/legal approval remains required.
- Malay remains `pending_human_review`; all final audio remains absent. Neither is accepted as a release-ready asset set.
- Any replacement must record source, creator, licence, permitted use, edit history, checksum where practical, and review date, then pass visual/audio, accessibility, import, and browser review.

## Required release record

Record commit/tag, Unity version, content/game version, exact test commands and fresh XML totals, build command/artifact inventory, browser matrix, LMS contract revision/environment, localization approvals, asset provenance, known limitations, reviewers, deployment/rollback owner, and date.

## Change classes

- Runtime/scene/bridge changes require focused RED/GREEN where applicable, broad EditMode/PlayMode, generated-scene checks, build, browser smoke, and independent review.
- Content/scoring/ID changes require graph, localization, scoring, compatibility, and human pedagogy review.
- Documentation-only changes require link/marker/heading/scope validation; do not invent runtime evidence.
- Asset changes require import checks, provenance, visual/audio, accessibility, and browser review.

## Human gates

Product/learning approves scope and rubric; the LMS owner approves the contract; privacy/security approves data/trust boundaries; the language reviewer approves Malay; accessibility and creative reviewers approve the experience; operations owns hosting, monitoring, and rollback; the release authority approves licences and makes the final decision.

## Version and rollback

Keep response IDs stable within a content version. Bump/version deliberately for incompatible semantics. Preserve a known-good WebGL artifact and configuration, make deployment atomic where possible, and test rollback before launch. Never claim a release from a local unbuilt checkout.
