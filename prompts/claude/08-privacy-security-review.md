# Review Privacy and Browser Security

## Objective

Threat-model launch/completion data and same-origin messaging, emphasizing child/learner data minimization and sanitized operations.

## Allowed Files

Read local LMS DTO/validators, `.jslib`, harness JavaScript, and `docs/12-PRIVACY-SECURITY.md`. Draft a report under `docs/` only if authorized.

## Files Not To Change

Do not change origin checks, add telemetry, inspect external systems/repositories, or handle real credentials/data. Never inspect or modify the external AGROVATOR LMS repository.

## Required Output

Return data-flow inventory, trust boundaries, abuse cases, mitigations, residual risks, and human privacy/security decisions.

## Required Tests

Specify/source-check wrong-origin, wrong-source, wrong-version, malformed request, stale callback, URL-secret, and logging cases; label unexecuted tests.

## Definition of Done

No production certification is claimed, sensitive values are absent, and each material risk has mitigation and owner.
