# Expand QA Edge Cases

## Objective

Find missing high-value tests across graph branching, timing, focus, retry, WebGL callbacks, localization, build, browser, and privacy boundaries.

## Allowed Files

Read tests, production sources, harness, docs 10-14 and acceptance status. Draft a ranked test plan or tests only within an explicitly granted file set.

## Files Not To Change

Do not modify production code during review, touch generated scenes/settings, inspect external repositories, or invent browser/build results. Never inspect or modify the external AGROVATOR LMS repository.

## Required Output

Return edge cases with precondition, action, observable assertion, layer, priority, and whether automation or human judgment owns it.

## Required Tests

If authorized to implement, use focused RED/GREEN and then relevant broad suites; otherwise provide executable cases and mark all unrun.

## Definition of Done

Cases are nonduplicative, project-specific, privacy-safe, and ordered by release risk with no fabricated pass claims.
