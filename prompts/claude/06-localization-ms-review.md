# Prepare Malay Localization for Human Review

## Objective

Help a qualified Malay reviewer replace English fallback copy while preserving meaning, terminology, key parity, and UI fit.

## Allowed Files

Read `en.json`, `ms.json`, scenario context, and localization/accessibility docs. Edit `ms.json` only with explicit human-reviewed translations supplied in the task.

## Files Not To Change

Do not invent approved Malay, change keys/IDs/status prematurely, alter code/scenes, or access external repositories. Never inspect or modify the external AGROVATOR LMS repository.

## Required Output

Produce a bilingual context sheet, terminology consistency checks, ambiguity questions, and reviewer sign-off record structure.

## Required Tests

Validate exact key parity, nonempty values, parser acceptance, escaped diagnostics, and in-context layout after approved edits.

## Definition of Done

Only human-approved Malay is committed, status truthfully reflects review, and unresolved linguistic questions remain explicit.
