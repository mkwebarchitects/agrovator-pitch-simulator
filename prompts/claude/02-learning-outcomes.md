# Align Learning Outcomes and Feedback

## Objective

Map the implemented scenario questions, competency tags, and result feedback to explicit formative learning outcomes without overstating efficacy.

## Allowed Files

Read `docs/03-GAMEPLAY-CONTENT.md`, `docs/07-SCORING-RESULTS.md`, scenario/localization JSON, and scoring tests. Draft changes only under `docs/` or localization/content paths named in the request.

## Files Not To Change

Do not alter runtime code, scenes, packages, scores, response IDs, or external repositories. Never inspect or modify the external AGROVATOR LMS repository.

## Required Output

Produce an outcome-to-question-to-feedback matrix, gaps, and bounded copy suggestions with confirmed/assumed labels.

## Required Tests

Check every cited key and response ID exists; if copy changes, validate English/Malay key parity and content parsing.

## Definition of Done

Every claim traces to implemented content, formative limits remain explicit, and learning-owner approvals are listed.
