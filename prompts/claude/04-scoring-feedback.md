# Audit Rubric and Result Feedback

## Objective

Assess whether authored deltas and feedback consistently support the seven-category formative rubric and the recovery mechanic.

## Allowed Files

Read scoring C#, tests, scenario JSON, localization, and `docs/07-SCORING-RESULTS.md`. Suggest changes; edit only content/docs explicitly authorized.

## Files Not To Change

Do not silently change caps, thresholds, IDs, completion schema, runtime code, scenes, or any external repository. Never inspect or modify the external AGROVATOR LMS repository.

## Required Output

Provide a response-by-category matrix, inconsistencies ranked by impact, proposed corrections, and assessment-owner decisions.

## Required Tests

Verify arithmetic against implemented caps/thresholds; require focused scoring/content tests for any accepted delta change.

## Definition of Done

Recommendations are reproducible, formative limits remain clear, recovery is rewarded consistently, and no metric is fabricated.
